# Step 11.1 — Init core integration

> Parent: [11-INIT.md](11-INIT.md) · SPEC ref: [init.ts](../SPEC-v2.md#inits), [persistence.ts](../SPEC-v2.md#persistencets), [Public API](../SPEC-v2.md#public-api)

## Goal

Create the v2 entry point and wire all step 1–10 modules into a single `initPageEditor()` function. Add selection persistence using `sessionStorage` (no lib-contentstudio dependency). Expose the v2 public API alongside the existing legacy exports so Content Studio can migrate to `initPageEditor` while legacy consumers still work. **No legacy code is deleted in this sub-step** — that is 11.2.

## persistence.ts

```ts
function initSelectionPersistence(): () => void;
```

- Reads `$config` for `contentId` and `fragment` (waits via `$config.subscribe` if config is not yet populated).
- Key = `pe-selected-path:<contentId>` in `sessionStorage`.
- `allowRootSelection = $config.fragment` — fragments may persist the root path.
- One-shot restore: read stored path, validate via `getRecord(path)`; if valid, `setSelectedPath(path)` + `getChannel().send({type: 'select', path})`. Scroll-into-view is handled by `SelectionHighlighter`, not here.
- Ongoing sync: `$selectedPath.listen(path => ...)` writes or removes the key.
- Returns an unsubscribe that disposes both the selection listener and (if used) the deferred config listener.

Reuses: `$config` / `$selectedPath` / `$registry` from `v2/state/`, `getChannel` from `v2/transport/channel.ts`, `root()` from `v2/protocol/path.ts`. **No `@enonic/lib-contentstudio` imports.**

## init.tsx

```ts
type RendererCallbacks = {
  onComponentLoadRequest?: (path: ComponentPath) => void;
};

type PageEditorInstance = {
  destroy: () => void;
  notifyComponentLoaded: (path: ComponentPath) => void;
  notifyComponentLoadFailed: (path: ComponentPath, reason: string) => void;
  requestPageReload: () => void;
};

function initPageEditor(
  root: HTMLElement,
  target: Window,
  callbacks?: RendererCallbacks,
): PageEditorInstance;
```

**Initialization sequence (in order):**

1. `createChannel(target)` → `setChannel(channel)`.
2. `document.body.classList.add('pe-overlay-active')`.
3. `const overlay = createOverlayHost(<OverlayApp />)`.
4. Declare `let currentDescriptors: DescriptorMap = {}` (cached for MutationObserver reuse).
5. `createAdapter(channel, { onPageState: (p) => { currentDescriptors = p.components; reconcilePage(root, currentDescriptors); }, onComponentLoadRequest: callbacks?.onComponentLoadRequest })`.
6. `initGeometryScheduler(path => getRecord(path)?.element)`.
7. `initHoverDetection()`.
8. `initSelectionDetection(channel)`.
9. `initKeyboardHandling(channel)`.
10. `initNavigationInterception(channel)`.
11. `initComponentDrag(channel)`.
12. `initContextWindowDrag(channel)`.
13. `initSelectionPersistence()`.
14. `startDomObserver(root, () => reconcilePage(root, currentDescriptors))`.
15. `channel.send({type: 'ready'})`.

**MutationObserver (`startDomObserver`):** watches `root` with `{childList: true, subtree: true}`. Filters mutations via `isEditorInjectedElement` (already exported from `v2/parse/emptiness.ts`). On meaningful mutation, calls `queueMicrotask(onReconcile)` to coalesce. Returns `() => observer.disconnect()`.

**`destroy()`:** run every `stop*` in reverse order, then:
- `destroyPlaceholders()` (from `v2/reconcile.tsx`).
- `overlay.unmount()`.
- `document.body.classList.remove('pe-overlay-active')`.
- Reset atoms: `setSelectedPath(undefined)`, `setHoveredPath(undefined)`, `setRegistry({})`, `setPageControllers([])`, `setLocked(false)`, `setModifyAllowed(true)`, `clearPageConfig()`, `resetDragState()`, `closeContextMenu()`.
- `resetChannel()`; if `Channel.destroy()` exists, call it.

**Messaging helpers:**
- `notifyComponentLoaded(path)` → `channel.send({type: 'component-loaded', path})`.
- `notifyComponentLoadFailed(path, reason)` → `channel.send({type: 'component-load-failed', path, reason})`.
- `requestPageReload()` → `channel.send({type: 'page-reload-request'})`.

All outgoing payloads already exist in `v2/protocol/messages.ts` — no protocol change required.

**Name collision**: `ComponentPath` is exported by both `v2/protocol` and the legacy line. Resolve by aliasing the legacy one — `export {ComponentPath as LegacyComponentPath} from '@enonic/lib-contentstudio/...'` — or by dropping the legacy re-export if no external consumer imports it via this package. Decide at implementation time after grepping downstream repos.
