# Step 11.2 — Legacy directory removal

> Parent: [11-INIT.md](11-INIT.md)

## Goal

Delete the legacy `page-editor/` and transitional `new-ui/` directories. Reduce the public API surface to v2 only. Leaves build config, dependencies, and preact/react aliases for later sub-steps.

**Blocker:** Content Studio must already call `initPageEditor()` in production. If any downstream consumer still imports `PageEditor`, `PageView`, `ItemView`, or the legacy `ComponentPath` from `@enonic/page-editor`, this sub-step cannot land.

## Scope

```
DELETE src/main/resources/assets/js/page-editor/   (~28 files)
DELETE src/main/resources/assets/js/new-ui/        (~50 files across 12 subdirs)
UPDATE src/main/resources/assets/js/index.ts       ← v2-only exports
UPDATE src/main/resources/assets/css/main.less     ← drop legacy imports/partials
UPDATE .storybook/preview.tsx                      ← drop any legacy imports (audit)
UPDATE stories (if any) referencing legacy modules
```

## index.ts (final form)

```ts
export * from './v2/protocol';
export {initPageEditor} from './v2/init';
export type {PageEditorInstance, RendererCallbacks} from './v2/init';
```

Drop `PageEditor`, `ItemView`, `ItemViewId`, `ItemViewIdProducer`, `CreateItemViewConfig`, `ItemViewFactory`, `ComponentItemType`, `ComponentView`, `DescriptorBasedComponentView`, `ContentBasedComponentView`, `PartComponentView`, `PartItemType`, `LayoutItemType`, `LayoutComponentView`, `TextItemType`, `TextComponentView`, `FragmentItemType`, `FragmentComponentView`, `RegionView`, `PageView`, `EditorEvents`, `EditorEvent`, `Element`/`NewElementBuilder` from `@enonic/lib-admin-ui`, legacy `ComponentPath` from `@enonic/lib-contentstudio`.

## main.less

Remove every `@import` that targets legacy selectors:
- lib-admin-ui imports (`variables.less`, `fonts.less`, `icons.less`, `menu.less`, `drag-helper.less`).
- lib-contentstudio imports (component item view context menu).
- Local partials coupled to legacy views (`highlighter.less`, `shader.less`, `drag-helper.less`, `drag-placeholder.less`, `item-view.less`, `image-view.less`, `part-view.less`, `text-view.less`, `region-view.less`, `page-view.less`).

Keep only the global `.pe-overlay-active` body rule if `initPageEditor` still needs it. If the file shrinks to that single rule, leave as-is (removing LESS toolchain is 11.4's job).

## Dependencies

Requires **11.1** to have landed so the v2 API is live, and Content Studio to have migrated.

## Verification

- `pnpm check` green.
- `pnpm build:dev` and `pnpm build:prod` succeed.
- `pnpm build-storybook` succeeds.
- `grep -r "from '.*page-editor/'" src/` returns nothing.
- `grep -r "from '.*new-ui/'" src/` returns nothing.
- Manual smoke in Content Studio with the v2 flow.
