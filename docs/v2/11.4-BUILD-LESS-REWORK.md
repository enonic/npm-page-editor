# Step 11.4 — Build and LESS rework

> Parent: [11-INIT.md](11-INIT.md)

## Goal

Simplify `vite.config.ts` now that jQuery and the lib-admin-ui/lib-contentstudio shims are gone. Decide the future of `main.less` — either shrink it to a minimal `.pe-overlay-active` rule or drop LESS entirely and remove the PostCSS chain.

**Blocker:** 11.3 must have landed — the build config simplifications depend on the dropped dependencies.

## Scope

```
UPDATE vite.config.ts
UPDATE src/main/resources/assets/css/main.less   ← shrink or delete
UPDATE package.json                              ← drop LESS/PostCSS deps if LESS is removed
UPDATE package.json scripts                      ← collapse build:dev:js/css into one target if feasible
```

## vite.config.ts

- Remove the `@rollup/plugin-inject` plugin (jQuery globals — no longer needed).
- Remove `resolve.alias` entries for `@enonic/lib-admin-ui` and `@enonic/lib-contentstudio`.
- Remove any JSX/TSX-only aliases that targeted legacy paths.
- If `main.less` is dropped: remove the CSS build target and collapse to a single entry. The current dual-mode setup (`BUILD_TARGET=js|css`) exists only to produce the CSS bundle alongside the JS bundle.

Keep:
- `preact/compat` aliases for `react`/`react-dom`/`react/jsx-runtime` (11.5 moves these to tsconfig but the Vite alias still resolves actual runtime imports).
- `@tailwindcss/vite` plugin.
- `vite-plugin-dts` (adjust `exclude` to drop `.xp/dev/*` entries).

## main.less

**Option A — keep a minimal file:** strip everything to a single `.pe-overlay-active { ... }` rule if that rule is still consumed. Keep the LESS toolchain for consistency.

**Option B (preferred if viable) — delete:** if `.pe-overlay-active` can be expressed in a tiny plain CSS file (or inlined via Tailwind inside the shadow root), delete `main.less` and every partial in `src/main/resources/assets/css/`.

## Deps to drop if LESS is removed

`devDependencies`:
- `less`
- `postcss`, `postcss-normalize`, `postcss-sort-media-queries`
- `cssnano`
- `autoprefixer`
- `@types/postcss-normalize`

Scripts:
- Replace `build:dev` / `build:prod` pnpm-parallel scripts with a single `vite build` call.
- Drop `build:dev:css` / `build:prod:css`.

## Dependencies

Requires **11.3** (dependency pruning).

## Verification

- `pnpm check` green.
- `pnpm build:dev` / `pnpm build:prod` produce the expected artifacts in `build/resources/main/assets/`.
- `pnpm build-storybook` succeeds.
- Manual smoke in Content Studio — overlay displays, shadow-DOM styling unaffected.
- `./gradlew build -Penv=dev` succeeds end-to-end (once in the release cycle).
