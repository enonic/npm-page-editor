# Step 11.5 — preact/compat → react alias

> Parent: [11-INIT.md](11-INIT.md)

## Goal

Move the `react` → `preact/compat` aliasing from Vite-only to TypeScript. Switch every v2 `from 'preact/compat'` import to `from 'react'` so the codebase matches `@enonic/ui` conventions and third-party typings resolve against standard React types.

**Blocker:** none strict — can land after 11.1 on its own. Sequenced here so `vite.config.ts` simplifications in 11.4 settle first.

## Scope

```
UPDATE tsconfig.app.json   ← add paths mapping
UPDATE tsconfig.test.json  ← add paths mapping
UPDATE src/main/resources/assets/js/v2/**        ← rewrite 'preact/compat' → 'react'
UPDATE .storybook/ tsconfig (if present)         ← mirror paths
```

## tsconfig.app.json / tsconfig.test.json

Add to `compilerOptions`:

```json
"baseUrl": ".",
"paths": {
  "react": ["./node_modules/preact/compat"],
  "react-dom": ["./node_modules/preact/compat"],
  "react/jsx-runtime": ["./node_modules/preact/jsx-runtime"],
  "react/jsx-dev-runtime": ["./node_modules/preact/jsx-dev-runtime"]
}
```

Evaluate flipping `jsxImportSource` from `"preact"` to `"react"` — with the alias above they resolve to the same module, but `"react"` produces standard `ReactNode` / `ReactElement` types that play better with third-party `.d.ts` files.

## Import migration

Rewrite in every v2 file that currently does `from 'preact/compat'`:

```ts
// before
import {useEffect, useState} from 'preact/compat';
import type {ReactNode} from 'preact/compat';

// after
import {useEffect, useState} from 'react';
import type {ReactNode} from 'react';
```

Known files (at time of writing, 8 hits):
- `v2/reconcile.tsx`
- `v2/components/SelectionHighlighter.tsx`
- `v2/hooks/use-page-descriptor-selector.ts`
- `v2/hooks/use-tracked-rect.ts`
- `v2/hooks/use-store.ts`
- plus 3 others surfaced by a final grep before the edit.

Final sweep: `grep -r "from 'preact/compat'" src/main/resources/assets/js/v2/` must return zero results.

## Dependencies

Can run independently of 11.2–11.4, but is sequenced after them so the codebase is fully v2 before API conventions change.

## Verification

- `pnpm check:types` green — no "Cannot find module 'react'" errors.
- `pnpm check:v2:lint` green.
- `pnpm build:dev` / `pnpm build:prod` produce identical runtime output to before (alias is a type-only change; Vite still resolves at runtime via `resolve.alias`).
- `pnpm build-storybook` succeeds.
- Manual smoke in Content Studio — no runtime regressions.
